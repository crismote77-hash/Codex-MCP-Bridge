# Contributing

Thanks for contributing!

## Basics
- Follow `AGENTS.md` / `CLAUDE.md` rules.
- Keep docs in sync with behavior.
- Use small, verifiable changes.

## Before opening a PR
- `npm run build`
- `npm test`
- `npm run lint`

## Runbook
Append a note with:
```bash
npm run runbook:note -- "What changed; verification; next step"
```
